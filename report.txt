  1. INTRODUCTION
================================================================================

  1.1 Background
  --------------
  The global smartphone market is one of the most competitive consumer
  electronics sectors, with over 1,000 new models released annually across
  dozens of brands. Prices range from under $100 to over $2,000, and the
  factors driving this wide range are often opaque to consumers, retailers,
  and new market entrants alike.

  Machine learning offers a systematic, data-driven way to decode these
  pricing dynamics. By training models on hardware specifications, we can
  learn the quantitative relationship between a phone's features and its
  fair market price — providing transparency that benefits multiple
  stakeholders.

  1.2 Problem Statement
  ---------------------
  Multiple groups face pricing-related challenges in the smartphone market:

    • Consumers cannot easily determine whether a listed price is fair
      relative to a phone's specifications, leading to overpaying.

    • Budget shoppers struggle to identify the best-value phones within
      their price range among hundreds of options.

    • Retailers risk losing sales by pricing too high or losing margin
      by pricing too low, without data to guide their decisions.

    • Startup OEMs (Original Equipment Manufacturers) entering the market
      have no objective, data-backed reference for setting competitive
      prices for new products.

    • Market analysts lack tools to systematically quantify brand premiums,
      regional pricing strategies, and specification–price relationships.

  1.3 Objectives
  --------------
  This project builds an end-to-end data science pipeline that:

    1. Predicts a fair market price for any smartphone from its hardware
       specifications using regression models.
    2. Classifies phones into intuitive price tiers (Budget, Mid-Range,
       Premium, Ultra-Premium) using classification models.
    3. Detects overpriced and underpriced phones to guide consumer decisions
       via a "Deal Detector" tool.
    4. Identifies which specifications drive smartphone pricing the most
       through feature importance and partial dependency analysis.
    5. Compares pricing across 5 countries to reveal regional price
       differences.
    6. Deploys an interactive Streamlit web application for non-technical
       users in two operating modes: Customer Mode and Company Mode.

  By applying regression, classification, and clustering techniques covered
  throughout ITCS 227, this project demonstrates a practical, end-to-end
  data science pipeline — from raw data to a deployed interactive tool.

  2.2 Dataset Description
  ------------------------
  The raw dataset contains smartphone listings from 2014 to 2025, covering
  19 brands (Apple, Samsung, Xiaomi, Oppo, Vivo, Honor, Google, etc.) with
  prices listed in 5 countries: USA, India, China, Pakistan, and Dubai.

  After removing 101 tablet entries (which follow different pricing patterns),
  829 smartphones remain for analysis.

    Feature                 Type     Example            Description
    ────────────────────────────────────────────────────────────────────
    Company Name            String   Apple              Brand / manufacturer
    Model Name              String   iPhone 16 128GB    Specific model
    Mobile Weight           String   174g               Weight in grams
    RAM                     String   6GB                Random Access Memory
    Front Camera            String   12MP               Selfie camera MP
    Back Camera             String   48MP               Main camera MP
    Processor               String   A17 Bionic         Chipset name
    Battery Capacity        String   3,600mAh           Battery size (mAh)
    Screen Size             String   6.1 inches         Display diagonal
    Launched Price (USA)    String   USD 799            USA launch price
    Launched Price (India)  String   INR 79,900         India launch price
    Launched Price (China)  String   CNY 5,999          China launch price
    Launched Price (Pak.)   String   PKR 249,999        Pakistan launch price
    Launched Price (Dubai)  String   AED 3,099          Dubai launch price
    Launched Year           Integer  2024               Year of release

  2.3 Data Quality
  ----------------
    • 0 missing values in the raw dataset after parsing.
    • All columns are initially stored as strings (e.g., "6GB", "3,600mAh").

  2.4 Preprocessing Steps
  ------------------------
  All string-encoded specifications were converted to numeric values using
  regular expression (regex) extraction:

    Step 1 — Numeric Parsing:
      • "6GB" → 6 (RAM in GB)
      • "3,600mAh" → 3600 (battery in mAh)
      • "174g" → 174 (weight in grams)
      • "6.1 inches" → 6.1 (screen size in inches)
      • "12MP" → 12 (camera resolution in MP)

    Step 2 — Multi-Camera Handling:
      • Entries such as "50MP + 12MP + 10MP" were parsed to extract:
          – Maximum resolution (primary lens MP)
          – Number of lenses (lens count)

    Step 3 — Price Normalization:
      • All country prices converted to USD using fixed approximate
        exchange rates for cross-country comparison.

    Step 4 — Feature Engineering:
      • Price-per-GB-RAM    = USD price / RAM (GB)
      • Camera Ratio        = Front Camera MP / Back Camera MP
      • Battery-per-Inch    = Battery (mAh) / Screen Size (inches)
      • Total Camera MP     = Sum of all camera lens resolutions
      • Interaction features: RAM × Total Camera MP, Screen × Battery

    Step 5 — Feature Binning:
      • RAM binned into quartiles using pd.qcut()
      • Battery capacity binned using pd.cut()
      • Price binned into 4 quartile-based tiers using pd.qcut()

    Final dataset used for modeling: 829 smartphones, 19 numeric features.

  3.2 Regression Modeling (Price Prediction)
  -------------------------------------------
  We trained 8 regression models to predict the USD launch price.
  All models use an 80/20 train-test split; continuous USD price is
  the target variable.

    Model                        Purpose
    ──────────────────────────────────────────────────────────────────
    Linear Regression            Baseline — simple, interpretable
    Ridge Regression (α=1)       Regularized — handles multicollinearity
    Poly-2 Ridge Regression      Polynomial Features (degree=2)
    Random Forest Regressor      Ensemble — captures non-linear patterns
    Gradient Boosting            Sequential ensemble — high accuracy
    HistGradientBoosting         Histogram-based boosting [Lab 07/10]
    Extra Trees Regressor        Randomized ensemble — reduces variance
    Tuned Random Forest          Hyperparameter-optimized RF

  Advanced evaluation techniques applied:
    • 5-Fold Cross-Validation (cross_val_score)
    • Learning Curves to diagnose overfitting vs. underfitting
    • Partial Dependency Plots for marginal feature effects
    • Model export using pickle for deployment

  Evaluation metrics: MAE, RMSE, R², CV R² (5-fold mean)

  "Deal Detector": Phones where |actual − predicted| > 25% of the
  predicted price are flagged as overpriced or underpriced.

  3.3 Classification (Price Tier Prediction)
  -------------------------------------------
  Phones were assigned to 4 price tiers based on USD launch price:

    Tier             Price Range
    ─────────────────────────────
    Budget           < $300
    Mid-Range        $300 – $700
    Premium          $700 – $1,000
    Ultra-Premium    > $1,000

  Three classification models were trained and compared:
    • Logistic Regression
    • Decision Tree Classifier
    • Random Forest Classifier

  Evaluation metrics: Accuracy, Precision, Recall, F1-Score,
  Confusion Matrix (visualized as a heatmap).

  3.4 Unsupervised Analysis (Clustering)
  ----------------------------------------
  Three clustering algorithms were trained and compared to discover
  natural groupings in the smartphone market without price labels:

    • K-Means (K=4, chosen via Elbow Method + Silhouette Score)
    • DBSCAN — density-based clustering with eps hyperparameter search
     
    • GMM (Gaussian Mixture Model) with n_components search

  All clustering was performed on standardized numeric features.
  Evaluation:
    • Silhouette Score for all methods
    • Hyperparameter grid search for optimal parameters
    • PCA for 2D cluster visualization
    • Cross-tabulation of discovered clusters vs. actual price tiers

  3.5 Cross-Country Price Analysis
  ----------------------------------
    • Currency normalization to USD using approximate exchange rates
    • Per-brand identification of cheapest and most expensive countries
    • Price markup heatmap (% difference from USA base prices)

  3.6 Deployment

  Python's pickle library and loaded by a Streamlit web application.

    – Customer Mode: User enters phone specs + listed price → app flags
      whether the phone is overpriced, underpriced, or fairly priced.
    – Company Mode: User enters phone specs → app returns the predicted
      fair market price for manufacturing/pricing decisions.

================================================================================
  5. DISCUSSION
================================================================================

  5.1 Interpretation of Results
  --------------------------------
    • R² = 0.91 means hardware specifications alone can explain 91% of
      the variance in smartphone market prices — a strong result that
      confirms the data-driven pricing approach is viable.
    • CV R² = 0.89 verifies the model generalizes to unseen phones well
      beyond the training set.
    • Learning curves show training and validation error converging,
      indicating a well-fitted model.
    • Weight being the top feature (importance 0.37) is intuitive:
      premium phones use heavier metal/glass construction, while budget
      phones use lighter plastic, making weight a strong proxy for build
      quality.
    • Battery capacity has lower importance than commonly assumed,
      suggesting consumers do not pay a significant premium purely for
      battery size.
    • The Deal Detector successfully flags phones priced ±25% from fair
      value, providing a practically useful consumer tool.
    • Classification accuracy of 80% is respectable but lower than the
      regression R² because predicting discrete tier boundaries is harder
      — phones near tier thresholds can legitimately belong to either
      neighboring category.

  5.2 Comparison with Proposal Expectations
  -------------------------------------------

    Metric                 Proposed Target    Achieved     Status
    ───────────────────────────────────────────────────────────────
    R² score               ~0.90              0.91         Met
    Classification acc.    ~90%               80%          Partially met
    Number of plots        16                 20           Exceeded
    Regression models      3                  8            Exceeded
    Clustering methods     1 (KMeans)         3            Exceeded

    Classification accuracy of 80% fell short of the 90% target because
    classifying phones into discrete tiers is harder than predicting
    continuous prices, especially without explicit brand encoding.

  5.3 Limitations
  ----------------
    • No processor / chipset encoding — chipset tier significantly affects
      price but model names cannot be directly encoded as numeric features
      without a separate benchmark database.
    • No storage variant feature — 128GB vs. 256GB configurations have
      different prices but storage capacity was not available in the dataset.
    • Static exchange rates — cross-country currency conversion uses fixed
      approximate rates rather than live data.
    • Dataset size — 829 phones is sufficient for this study but larger
      datasets would improve generalizability.
    • Brand effect not fully captured — brand premium is partially captured
      through weight/specs but not encoded explicitly in all models.

  5.4 Potential Future Work
  --------------------------
    1. Add SHAP values for individual-prediction explainability.
    2. Extract processor benchmark scores and storage size as features.
    3. Integrate live exchange rate APIs for real-time price comparison.
    4. Add time-series forecasting for smartphone price trend prediction.
    5. Deploy the Streamlit application online via Streamlit Cloud.
    6. Expand the dataset with 2025–2026 smartphone releases.
